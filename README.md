Welcome to the **BNB AI Coding Sprint** Open Track! This is where the community shapes the hackathon by proposing ideas for builders to work on.

## üéØ What is the Open Track?

The Open Track is a community-driven initiative where anyone can submit project ideas for hackathon participants to build. If your idea gets picked up and built, **you'll be credited as the idea originator**.

## üìã How to Submit an Idea

### Step 1: Check Existing Ideas
Browse the [Issues](https://github.com/tari404/ideas/issues) tab to see what's already been proposed. Avoid duplicates!

### Step 2: Create a New Issue
Click **["New Issue"](https://github.com/tari404/ideas/issues/new/choose)** and select the **"üí° Idea Submission"** template.

### Step 3: Fill Out the Template
Complete all required fields:

| Field | Description |
|-------|-------------|
| **Idea Title** | Clear, concise name for your idea |
| **Track** | Agent / DeFi / Consumer Apps / Builder Tools / Infrastructure |
| **Problem Statement** | What problem does this solve? |
| **Proposed Solution** | High-level description of the solution |
| **Why BNB Chain?** | Why should this be built on BNB Chain? |
| **Success Criteria** | How do we know if it's built successfully? |
| **Resources** | Any helpful links, references, or documentation |

### Step 4: Engage
- Respond to questions and feedback on your idea
- Help refine the scope if needed
- Share on Twitter with **#VibingOnBNB** to gain visibility

## ‚úÖ Submission Rules

1. **One idea per issue** - Keep submissions focused
2. **Be specific** - Vague ideas won't get picked up
3. **Open source first** - Ideas should result in open-source contributions
4. **BNB Chain native** - Ideas must be relevant to BNB Chain ecosystem
5. **Verifiable deliverables** - Should result in runnable demos or working code
6. **No duplicates** - Search existing issues before submitting
7. **English only** - For accessibility across the global community
